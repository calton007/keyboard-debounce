# KeyboardDebounce Design

## 目标

KeyboardDebounce 是一个 Windows 用户态键盘防抖托盘程序，用低层键盘钩子缓解机械或笔记本键盘的异常短间隔重复触发。

## 防抖策略

- 每个虚拟键码维护独立学习状态。
- 同一键在有效阈值内再次 `KeyDown` 时拦截。
- 明显长按超过 `LongHoldBypassMs` 后放行，避免破坏退格、方向键和普通长按输入。
- 全局敏感度只影响运行时有效阈值，基础阈值仍按键保存。

## 自学习

- 接近阈值的拦截事件会提高该键阈值。
- 连续疑似抖动但被放行的事件会小幅提高阈值。
- 长时间稳定输入才缓慢降低阈值。
- 最近调整量、原因和时间写入学习状态，便于诊断学习是否发生。

## 忽略键

- 忽略列表按虚拟键码保存。
- 忽略键完全放行，不参与拦截、不创建学习状态、不更新统计。
- 表格仍显示已忽略键，方便取消忽略。

## 设置表格

- 启动时默认打开设置窗口；用户勾选“静默运行”后，下次启动只进入托盘。
- `VK` 排序时所有行按 VK 正常排序。
- `阈值(ms)` 显示按全局敏感度换算后的实际运行阈值；基础学习阈值仍保存在学习状态中。
- 非 `VK` 排序时未忽略项始终在前，忽略项始终在后，升序和降序都不反转这个分组规则。
- 自动刷新只原地更新已有行数值，不重建表格、不改变排序、不改变当前焦点。
- 新按键出现后需要手动刷新才会加入表格。

## 隐私和边界

- 用户配置保存到 `%APPDATA%\KeyboardDebounce\settings.json`。
- 自动学习状态保存到 `%APPDATA%\KeyboardDebounce\learning-state.json`。
- 两个文件只保存虚拟键码、阈值、计数、时间和学习原因，不保存输入文本或文本序列。
- 程序是用户态方案，不覆盖登录界面、UAC 安全桌面或所有管理员权限窗口。
- 低层钩子不能稳定区分内置键盘和外接键盘，因此忽略规则对所有键盘全局生效。
